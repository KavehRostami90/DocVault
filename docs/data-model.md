# Data Model

## Aggregates

### `Document`

Core aggregate representing a stored file and its extracted content.

| Column | Type | Notes |
|---|---|---|
| `Id` | `uuid` | `DocumentId` value object wrapping a GUID |
| `OwnerId` | `uuid?` | FK to `AspNetUsers.Id`; `null` for documents created before auth was added (admin-only visible) |
| `Title` | `text` | 1–256 characters; required |
| `FileName` | `text` | No path traversal (`..`, `/`, `\`) |
| `ContentType` | `text` | `text/plain`, `application/pdf`, `application/vnd.openxmlformats…` |
| `Size` | `bigint` | 1 byte – 50 MB |
| `Hash` | `text` | SHA-256 hex; unique constraint |
| `Text` | `text` | Extracted plain text; populated by `IndexingWorker` |
| `Embedding` | `vector(768)` | 768-dimension float vector; set by `EmbeddingStage`; HNSW index (`vector_cosine_ops`) for ANN search |
| `SearchVector` | `tsvector` | Computed column: `to_tsvector('english', Title \|\| ' ' \|\| coalesce(Text,''))`; GIN index for full-text search |
| `Status` | `int` | `Pending=0`, `Imported=1`, `Indexed=2`, `Failed=3` |
| `IndexingError` | `text?` | Populated when `Status = Failed` |
| `CreatedAt` | `timestamptz` | Set on insert; never updated |
| `UpdatedAt` | `timestamptz` | Bumped on every `Touch()` |

**Status transitions:**
```
Pending → Imported (MarkImported  → raises DocumentImported event)
Imported → Indexed (MarkIndexed   → raises DocumentIndexed event)
Imported → Failed  (MarkFailed(error?) → stores IndexingError)
```

### `Tag`

| Column | Type | Notes |
|---|---|---|
| `Id` | `uuid` | |
| `Name` | `text` | 1–50 chars; lowercase alphanumeric + hyphens/underscores |

Many-to-many with `Document` via the `DocumentTags` join table. A document may have at most 20 tags.

### `ImportJob`

Process ticket for background indexing. Linked to the `Document` it is indexing.

| Column | Type | Notes |
|---|---|---|
| `Id` | `uuid` | |
| `DocumentId` | `uuid` | FK to `Documents.Id`; required |
| `FileName` | `text` | Original upload file name |
| `StoragePath` | `text` | Path within `IFileStorage` to the raw binary |
| `ContentType` | `text` | MIME type used to select the correct `ITextExtractor` |
| `Status` | `int` | `Pending=0`, `InProgress=1`, `Completed=2`, `Failed=3` |
| `StartedAt` | `timestamptz` | Set at construction |
| `CompletedAt` | `timestamptz?` | Set by `MarkCompleted()` and `MarkFailed()` |
| `Error` | `text?` | Exception message on failure |

**Status transitions:**
```
Pending → InProgress → Completed
                     → Failed (stores Error)
```

### `IndexingQueueEntries` (PostgresWorkQueue)

## Authentication Tables

These tables are managed by **ASP.NET Core Identity** and the custom auth code in `DocVault.Infrastructure/Auth/`.

### `AspNetUsers` (`ApplicationUser`)

Extends `IdentityUser<Guid>` with three extra columns.

| Column | Type | Notes |
|---|---|---|
| `Id` | `uuid` | Primary key |
| `Email` | `text` | Unique; used as login name |
| `PasswordHash` | `text` | BCrypt-hashed; managed by Identity |
| `DisplayName` | `text?` | Optional friendly name |
| `IsGuest` | `bool` | `true` for ephemeral guest accounts (24h TTL) |
| `CreatedAt` | `timestamptz` | Set at registration |
| + standard Identity columns | | `NormalizedEmail`, `ConcurrencyStamp`, `SecurityStamp`, etc. |

### `RefreshTokens`

| Column | Type | Notes |
|---|---|---|
| `Id` | `uuid` | Primary key |
| `UserId` | `uuid` | FK to `AspNetUsers.Id` (cascade delete) |
| `Token` | `text` | Opaque random Guid; unique |
| `ExpiresAt` | `timestamptz` | 7 days for users; 24 hours for guests |
| `CreatedAt` | `timestamptz` | |
| `RevokedAt` | `timestamptz?` | Set on logout or token rotation |
| `ReplacedByToken` | `text?` | Token that replaced this one (rotation chain) |

### Standard ASP.NET Identity Tables

`AspNetRoles`, `AspNetUserRoles`, `AspNetUserClaims`, `AspNetRoleClaims`, `AspNetUserLogins`, `AspNetUserTokens` — generated by Identity migrations, not customised.

**Roles:** `Admin`, `User`, `Guest`

### Ownership

`Documents.OwnerId` (nullable `uuid`) links a document to its owner. Rules enforced in handlers:

- Regular users (`User` / `Guest`) only see and operate on their own documents.
- `Admin` users bypass ownership filtering and see all documents.
- Documents with `OwnerId = null` (created before auth was introduced) are visible to admins only.

## Domain Events

| Event | Raised by | Payload |
|---|---|---|
| `DocumentImported` | `Document.MarkImported()` | `DocumentId` |
| `DocumentIndexed` | `Document.MarkIndexed()` | `DocumentId` |
| `SearchExecuted` | `SearchDocumentsHandler` | query, result count |

Events are collected on the aggregate (`AggregateRoot<TId>.DomainEvents`) and dispatched via `IDomainEventDispatcher` after the unit-of-work commits.

## Database Migrations

Migrations live in `src/DocVault.Infrastructure/Migrations/`. The API applies pending migrations automatically on startup, so you rarely need to run them manually.

To apply manually (ensure the `db` container is running first):

```bash
docker compose up -d db

# Use the credentials from your .env file
dotnet ef database update \
  --project src/DocVault.Infrastructure \
  --startup-project src/DocVault.Api \
  -- --connectionString "Host=localhost;Port=5432;Database=docvault;Username=docvault;Password=<DOCVAULT_DB_PASSWORD>"
```

To add a new migration:

```bash
dotnet ef migrations add <MigrationName> \
  --project src/DocVault.Infrastructure \
  --startup-project src/DocVault.Api
```
